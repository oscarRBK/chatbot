Widget wired to the real Make webhook. First message sends no session_id; every reply returns one and the next message reuses it so the backend keeps context. Open the launcher bottom-right ↘

import { Chatbot } from 'provider-chatbot';

Chatbot.init({
  title: 'Rubrika',
  subtitle: 'Asistente IA',
  welcomeMessage: '¡Hola! ¿En qué puedo ayudarte?',
  placeholder: 'Escribe tu mensaje…',
  provider: 'make',
  endpoint: 'https://hook.eu1.make.com/y9gckk9zey1gjtu5kdp1ai9au27svewa',
  consent: true,                 // pass your real consent state
  // utm_* and page_url/referrer are read from the page automatically
  onSession: ({ sessionId, conversationId }) =>
    console.log('session', sessionId, conversationId),
});
